Selenium automates browsers. That's it!

What you do with that power is entirely up to you.

Primarily, it is for automating web applications for testing purposes, but is certainly not limited to just that.

Boring web-based administration tasks can (and should!) be automated as well.

Getting Started

  • Selenium WebDriver

  • You want to create robust, browser-based regression automation suites and tests, scale and distribute scripts across many environments

    Then you want to use Selenium WebDriver

  • Selenium IDE

  • You want to create quick bug reproduction scripts create scripts to aid in automation-aided exploratory testing Simple record-and-playback of interactions with the browser.

    Then you want to use Selenium IDE.

  • Selenium Grid

  • You want to take Selenium Remote Control to another level by running tests on many servers at the same time & cutting down on the time it takes to test multiple browsers or operating systems.

    Then you want to use Selenium Grid.
